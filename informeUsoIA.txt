3. REFLEXIÓN SOBRE EL USO DE LA IA
----------------------------------------------------------------
La inteligencia artificial fue utilizada como herramienta de
apoyo al desarrollo, no como sustituto del análisis propio.

El proceso de trabajo fue el siguiente:

a) ANÁLISIS PROPIO PREVIO:
   Antes de escribir cualquier prompt, se leyó y analizó el
   cuadernillo del examen para identificar los requisitos de
   negocio: las reglas de tarifas por peso, el flujo de estados
   unidireccional, el límite de 3 intentos de entrega, el
   descuento por NIT y la estructura del código de rastreo.

b) USO DE IA PARA ACELERAR EL DESARROLLO:
   Con los requisitos claros, se usó la IA para generar el
   código base de la API en C# con la arquitectura en capas
   (Controllers → Services → Data), lo cual habría tomado
   varias horas hacerlo manualmente desde cero.

c) CORRECCIONES IDENTIFICADAS Y APLICADAS:
   - La versión de .NET debió ajustarse de 8 a 9 según el
     entorno disponible en la máquina local.
   - El archivo de pruebas omitió el "using Xunit;" que es
     necesario para que los atributos [Fact] y [Theory] sean
     reconocidos por el compilador.
   - El push inicial a GitHub falló porque el repositorio fue
     creado con un README automático, requiriendo --force.

d) VALIDACIÓN PROPIA:
   Cada historia de usuario fue revisada contra la rúbrica del
   examen para verificar que cubría los criterios de: formato,
   criterios medibles y cobertura de todos los requerimientos.
   Se verificó que los 7 endpoints cubrían todos los flujos
   descritos en el cuadernillo.


4. APRENDIZAJES OBTENIDOS
----------------------------------------------------------------
- El diseño de APIs REST requiere pensar en los casos borde:
  transiciones de estado inválidas, datos faltantes, división
  por cero en reportes cuando no hay datos.

- Las pruebas unitarias con base de datos en memoria (InMemory)
  permiten probar la lógica de negocio de forma aislada sin
  necesitar una base de datos real instalada.

- Render.com con Docker simplifica significativamente el
  despliegue de aplicaciones .NET sin necesidad de configurar
  un servidor manualmente.

- El uso responsable de IA implica entender el código generado,
  no solo copiarlo. Cada corrección de error requirió
  comprender qué estaba fallando y por qué.
